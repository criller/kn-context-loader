## 文档分册

- `docs/01-概念约定/`：术语、接口定义与示例数据
- `docs/02-场景分析/`：场景拆解、能力说明与用例表
- `docs/03-需求管理/`：统一检索接口需求规格与模板
- `docs/04-场景交互分析/`：澄清与交互策略分析
- `docs/05-方案设计/`：流程设计、意图模板与方案讨论
- `docs/06-实现方案/`：实现思路与提示词材料（不等同于代码实现）
- `docs/07-草稿方案/`：架构草稿、数据流与设计草案

## 使用与协作约定

- 仓库定位：仅管理文档与资料，不包含运行代码
- 命名规范：延续现有分册编号前缀（如 `01-概念约定`）与清晰标题
- 引用规范：尽量使用相对路径引用同仓库文档与图片
- 评审与沉淀：
  - 文档初稿可提交为草稿（建议在首段标注更新时间与状态）
  - 经过讨论与评审后再沉淀为稳定版本
