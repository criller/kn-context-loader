# V2 澄清前提与适用场景（工具优先）

定位与前提
- 工具为核心：我们交付的是“好用、可信、可控规模”的业务知识网络检索工具，澄清只是配套机制。
- 目的：
  - 降误检：避免选错实体/方向造成结果偏离。
  - 守规模：防止结果爆炸与延迟/配额风险。
  - 统一口径：避免统计/法律口径不一致。
  - 满足合规：在敏感范围前置确认。
  - 引导迭代：结果后用自然语言建议下一步。

澄清使用边界（方式）
- 默认（不澄清）：先执行安全规模结果，结果页提供轻提示（只看在营、近三年、扩展到两跳、按时间排序、多给一些）。
- 软澄清（提示）：返回结果后提示用户补充条件或调整范围；不暴露技术参数。
- 阻断澄清（必须）：执行前必须确认，仅在高风险场景（严重歧义、极端规模、合规、矛盾、不可降级能力越界）。

适用场景（何时需要澄清）
- 名称/实体歧义：候选接近时默认展示最可信并允许更换；严重接近且误选影响语义→阻断。
- 关系方向/类型未指明：默认常用方向（企业→园区），结果页提示可切换；语义差异极大且任务只允许一种→阻断。
- 规模与深度风险：超安全阈值先按默认阈值执行并提示；极端规模（如>2000）→阻断或分步。
- 过滤缺失：基数大且未指定过滤，先按默认口径执行（近三年/相关性），再提示；合规必须明确→阻断。
- 字段别名/口径差异：默认采用标准字段，提示可查看别名或历史；统计/法律口径差异→阻断确认。
- 能力越界：降级为更小子问题，提示能力边界；不可降级且语义明确→阻断。
- 合规/风控：敏感范围或必须确认→阻断。
- 输入矛盾或不一致：互斥条件→阻断。

默认阈值建议（可按业务调优）
- 列表规模：≤30安全、31–200软澄清、>2000阻断；图规模：≤50安全、51–500软澄清、>2000阻断。
- 深度：>1软澄清（先返回一跳并提示扩展），>3且规模高可能阻断。
- 名称歧义阻断：top1<0.50且top1-top2≤0.05；否则软澄清。

在交互手册中的使用方式（统一模板）
- 每个场景均包含：
  - 最小可用检索策略（默认对象/排序/时间窗口/深度/数量）
  - 是否澄清：默认/软澄清/阻断澄清（引用本前提与阈值）
  - 澄清话术（自然语言）与Agent参数模板（隐藏于元数据）
  - 守护与自动降级提示（透明解释）
  - 验收用例与体验指标（默认执行可用性、阻断比例、可读性、透明解释率）

示例自然语言微文案（结果页一键操作）
- “只看在营” / “近三年” / “只看园区” / “扩展到两跳” / “按时间排序” / “多给一些”。
- 组合示例：“只看江苏并按时间排序，最多一百条”。
