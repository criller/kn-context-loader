# 场景：实例检索（Entity Lookup）

目标（聚焦用户与业务）
- 通过名称/别名/类型/属性筛选，定位一个或多个对象实体（企业、园区、专利、物质物料、反应、工艺等）。
- 结果用于确认存在性、查看基础信息，或作为后续分析（如子图/关系判断）的起点。

用户故事（化工示例，可替换为你们领域）
- SC-001 企业名称检索
  - “查找名称包含‘化工’的企业”
  - TBox对象/关系路径：对象类型 enterprise（直接实体检索，无关系路径）
  - 期望：返回企业列表及基础信息（名称、类型、地址、简介等）。
- SC-002 园区检索
  - “有哪些名字里带‘开发区’的园区？”
  - TBox对象/关系路径：对象类型 park（直接实体检索，无关系路径）
  - 期望：返回园区列表及名称、地址等信息。
- SC-003 专利号/标题检索
  - “专利号为‘CN1234567’的是哪条？”、“标题含‘乙醇’的专利”
  - TBox对象/关系路径：对象类型 patent（直接实体检索，无关系路径）
  - 期望：返回专利的编号、标题、状态、IPC、日期、摘要等。
- SC-004 物质物料属性检索
  - “CAS号是 64-17-5 的物质是什么？”、“危化品类别为易燃液体的物料有哪些？”
  - TBox对象/关系路径：对象类型 material（直接实体检索，无关系路径）
  - 期望：返回物料名称、CAS、化学式、危化品类别、储存要求等。
- SC-005 反应与工艺检索
  - “有无名为‘酯化反应’的反应？”、“查找包含‘精馏塔’设备的工艺”
  - TBox对象/关系路径：对象类型 reaction / process（直接实体检索，无关系路径）
  - 期望：返回反应/工艺的名称、描述、关键属性。

基础能力（文本版）
- 支持按名称/别名/类型/属性进行检索；支持模糊匹配与同义词归一；
- 支持结果限制与分页（limit）；支持按相关性或时间进行排序；
- 支持字段级过滤（如 status、ipc、hazard_class 等）；
- 能区分不同对象类型并返回基础属性集合。

注意事项
- 同名实体较多时需提示进一步限定（对象类型、地域/地址关键字等）。
- 对字段别名进行归一，降低术语差异的影响。
- 这些检索结果可作为后续子图查询的种子实体。
