# 场景：文本检索 / RAG（Texts）【M2】

目标（聚焦用户与业务）
- 检索与实体相关的文本字段（摘要、描述、流程、安全说明等），用于问答或辅助说明，提升可读性与可解释性。

用户故事（化工示例，可替换为你们领域）
- TX-001 专利摘要检索
  - “检索包含‘乙醇氧化’的专利摘要”
  - 按TBox对象/属性：对象类型 patent，文本字段 patent.abstract
  - 期望：返回匹配的专利与摘要片段。
- TX-002 工艺说明文本检索
  - “查找包含‘精馏塔’的工艺说明”
  - 按TBox对象/属性：对象类型 process，文本字段 process.flow / process.equipment / process.safety_notes
  - 期望：返回匹配工艺及相关说明片段。
- TX-003 企业简介/公告
  - “企业A的简介和近期公告”
  - 按TBox对象/属性：对象类型 enterprise，文本字段 enterprise.brirf_info（示例字段名）
  - 期望：返回企业简介或文本说明。
- TX-004 物料安全说明
  - “乙醇的储存要求与安全注意事项”
  - 按TBox对象/属性：对象类型 material，文本字段 material.storage_requirements；（可关联工艺的 process.safety_notes）
  - 期望：返回物料的储存与安全说明。
- TX-005 反应描述检索
  - “与酯化相关的反应描述”
  - 按TBox对象/属性：对象类型 reaction，文本字段 reaction.description / reaction.conditions
  - 期望：返回反应的描述/条件说明。

基础能力（文本版）
- 支持指定对象类型与文本字段的检索；
- 支持关键词/短语匹配与别名归一；
- 支持结果片段（snippet）与高亮（文本版描述）。

注意事项
- 需明确文本字段口径（来自对象属性，不涉及接口实现）；
- 如需先通过关系构造集合后再进行文本检索，请明确方向（正向/反向/双向）；默认采用正向。
- 文本匹配可能存在歧义与噪声，建议配合澄清或过滤；
- 结果规模与片段长度需受控（limit）。
