# 场景：属性获取（Attribute Fetch）

目标（聚焦用户与业务）
- 获取对象实体的属性值；支持指定条件（如最新值、状态筛选）。
- 用于回答“是什么/值是多少/有哪些”等问题，常与实例检索/子图查询结合。

用户故事（化工示例，可替换为你们领域）
- AT-001 企业地址
  - “企业A的注册地址是什么？”
  - TBox对象/关系路径：对象类型 enterprise（直接读取 entity.address）
  - 期望：返回企业的 address 与相关基础信息。
- AT-002 专利状态与日期
  - “专利号 CN1234567 的当前状态与授权日期？”
  - TBox对象/关系路径：对象类型 patent（直接读取 patent.status、grant_date）
  - 期望：返回 patent.status、grant_date 等字段。
- AT-003 物料属性
  - “乙醇的化学式与储存要求是什么？”
  - TBox对象/关系路径：对象类型 material（直接读取 material.formula、storage_requirements）
  - 期望：返回 material.formula、storage_requirements 等。
- AT-004 （关系属性为未来增强）
  - “若模型支持关系边属性（例如 企业→园区 的时间戳/备注），则可读取相应字段。”
  - 当前说明：暂不支持关系边属性读取，待后续模型扩展。

基础能力（文本版）
- 支持实体属性读取；暂不支持关系边属性读取（未来增强）。
- 支持字段别名归一与缺失值处理；
- 支持条件限定（最新值、状态、时间范围等）。

注意事项
- 属性名可能存在别名差异（如“注册地址/地址”），需归一；
- 当属性不存在或为空时，需清晰告知并避免误导。
