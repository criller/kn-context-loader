# 术语约定
## 1、总术语约定
1、业务知识网络（Business Knowledge Network），业务知识网络是一个知识表示模型，用于描述业务领域的概念、属性、关系等。业务知识网络分为概念层（tbox）和数据层（abox）。
2、在业务知识网络本体模型中，概念层（tbox）用于描述业务领域的概念、属性、关系等，数据层（abox）用于存储具体的业务数据实例。
3、在业务知识网络中，概念上分为对象类、关系类，对应知识图谱中的节点（node）和边（edge）。

## 2、概念层术语约定
2.1 概念（Concept）：业务知识网络中的基本元素，用于描述业务领域的概念、属性、关系等。所以概念层的统称叫概念，具体的概念有对象类和关系类、对象类的属性和关系类的映射关系。这些都是概念。
2.2 对象类（Object Class）：业务知识网络中的概念，用于描述业务领域的实体（如客户、订单、产品等）。对象类在知识图谱中对应节点（node）。对象类会有属性（attribute），用于描述对象的特征（如客户姓名、订单号、产品名称等）。
2.3 关系类（Relationship Class）：业务知识网络中的概念，用于描述业务领域的关系（如客户订单关系、产品分类关系等）。关系类在知识图谱中对应边（edge）。关系类主要是维护两个对象类之间的关系，每个关系实例都有一个开始节点（source node）和一个结束节点（target node）。以及两个对象类之间的映射关系，是通过哪些属性建立关联的。

## 3、数据层术语约定
3.1 数据实例（Data Instance）：业务知识网络中的具体业务数据实例，用于存储业务领域的实体和关系的具体值。数据实例在知识图谱中对应节点（node）和边（edge）。

实例是表示业务知识网络中的具体业务数据，是数据层的基本元素。
3.2 对象实例：在概念层叫对象类，在数据层叫对象实例
